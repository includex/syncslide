# 모바일 리모컨 UX 요건 (Remote Controller)

> `/remote/[sessionId]` 화면의 모바일 최적화 정책
> 기존 PRD §7.4, §11을 보완하는 리모컨 인터랙션 상세 스펙

---

## 1. 기본 방향

- 리모컨은 **모바일 브라우저 전용** 화면이다.
- **가로 모드(Landscape)가 기본**이다.
- 세로 모드(Portrait)는 선택적으로 지원한다 (기기를 돌리면 자동 전환).

---

## 2. 레이아웃 구조 (가로 모드 기준)

```
┌──────────────────────────┬──────────┐
│                          │          │
│       슬라이드 영역       │  스크립트 │
│       (좌측 ~80%)        │  레이저   │
│                          │  판서     │
└──────────────────────────┴──────────┘
                              (우측 20%)
```

- **슬라이드 영역**: 화면 좌측 약 80% 차지
- **버튼 패널**: 화면 우측 약 20%, 3개 버튼이 세로로 **영역을 꽉 채워** 균등 배치
  - 버튼은 **네모(rectangle)** 형태, border-radius 없음 또는 최소화 (4px)
  - 각 버튼이 우측 패널 높이의 1/3씩 차지
  - 버튼 사이 구분선: Dark Border `#2f2b4a` 1px
- 버튼 패널 배경: Dark Surface `#1c1a2c`

---

## 2-1. 세로 모드 레이아웃 (Portrait)

세로 모드는 `@media (orientation: portrait)`로 분기한다. **모드 종류·전환 동작·버튼 스타일은 가로 모드와 완전히 동일**하며, 패널의 배치 방향만 바뀐다.

```
┌────────────────────────────┐
│        (Dark Base 여백)     │
│   ┌────────────────────┐    │
│   │    슬라이드 영역     │    │ ← 상단: 16:9 비율 유지, 중앙 정렬
│   │    (16:9 비율)      │    │
│   └────────────────────┘    │
│        (Dark Base 여백)     │
├──────────┬─────────┬────────┤
│ 스크립트  │  레이저  │  판서   │ ← 하단: 버튼 3개 가로 1/3 균등
└──────────┴─────────┴────────┘
```

**기본 구조**

- **상단 — 슬라이드 영역**: 16:9 비율을 유지한 채 영역 중앙에 정렬한다. 비율 때문에 생기는 남은 공간(레터박스)은 Dark Base `#0f0e17` 배경으로 채운다.
- **하단 — 버튼 패널**: 스크립트 / 레이저 / 판서 3개 버튼을 **가로로** 배치하고, 각 버튼이 너비의 **1/3씩 균등 분할**로 꽉 채운다.
  - 버튼 사이 구분선: Dark Border `#2f2b4a` 1px (가로 모드의 가로 구분선이 세로 구분선으로 바뀜)
  - 버튼 패널 배경: Dark Surface `#1c1a2c`

**버튼 스타일**: 가로 모드와 동일 — 네모(border-radius 4px), 텍스트만, 비활성 `#1c1a2c`/`#9ca3af` · 활성 `#7270ff`/`#ffffff`.

**모드 전환 동작**: 가로 모드와 완전히 동일 (§4 참조). 슬라이드 넘기기·레이저·판서·스크립트 인터랙션 모두 동일하게 동작한다.

**방향 전환 시 상태 유지**

- 화면 방향이 바뀌어도 **현재 모드 상태를 유지**한다. 예: 레이저 모드에서 기기를 돌리면 전환 후에도 레이저 모드가 유지된다.
- 판서 Canvas 내용도 유지된다 (§7).

**판서 모드는 가로 단일 레이아웃**

- 판서 모드는 세로 전용 레이아웃을 제공하지 않고, **방향과 무관하게 가로 모드와 동일한 단일 판서 화면**(슬라이드 전체 채움, 우측 패널 숨김)을 사용한다.
- 세로 상태에서 판서 버튼을 누르면 가로 모드와 동일한 판서 모드로 전환되며, "가로로 돌려서 판서하세요" 안내 토스트를 표시한다.

**슬라이드/스크립트 모드 — 넘기기 영역(세로 모드)**

- 세로 모드에서 좌우 스와이프·좌우 탭으로 슬라이드를 넘기는 영역은 **하단 버튼 패널을 제외한 슬라이드 영역 전체**다. 16:9 슬라이드 위·아래의 Dark Base 여백(레터박스)을 탭/스와이프해도 페이지가 넘어간다.
- 스크립트 모드도 동일하게 좌우 스와이프·탭으로 이전/다음 장의 스크립트를 볼 수 있으며, 페이지 번호는 슬라이드/스크립트 모드 모두 **슬라이드 영역 하단 중앙**에 표시한다.

## 3. 4가지 모드

### 3.1 슬라이드 모드 (디폴트)

**레이아웃**
- 슬라이드 영역 + 우측 버튼 패널 (스크립트 / 레이저 / 판서)
- dim 오버레이는 적용하지 않는다 (슬라이드를 밝게 그대로 표시)

**슬라이드 넘기기**
- **전체 화면 좌우 스와이프** → 이전/다음 슬라이드
- **슬라이드 영역 좌측 탭** → 이전 슬라이드
- **슬라이드 영역 우측 탭** → 다음 슬라이드

**버튼 상태**
- 스크립트 / 레이저 / 판서 버튼 모두 비활성 (기본 색상)

---

### 3.2 스크립트 모드

**진입**: 슬라이드 모드에서 스크립트 버튼 탭
**종료**: 스크립트 버튼 다시 탭 → 슬라이드 모드 복귀

**레이아웃**
```
┌──────────────────────────┬──────────┐
│                          │ [스크립트]│ ← 활성
│   자막 텍스트 전체 표시   │  레이저   │
│   (슬라이드 이미지 없음)  │  판서     │
│   현재 페이지 번호만 표시 │          │
└──────────────────────────┴──────────┘
```

- 슬라이드 이미지 **숨김** — 좌측 영역 전체를 자막 텍스트로 채움
- 현재 몇 번째 슬라이드인지 페이지 번호만 상단에 작게 표시 (예: `2 / 5`)
- 자막 텍스트: 큰 폰트로 가독성 우선 (권장 24px 이상, weight 500)
- 텍스트 색상: Dark Text Primary `#f9fafb`
- 배경: Dark Base `#0f0e17`
- 스크립트 버튼 → 활성 색상 (Electric Violet `#7270ff`)

---

### 3.3 레이저 모드

**진입**: 슬라이드 모드에서 레이저 버튼 탭
**종료**: 레이저 버튼 다시 탭 → 슬라이드 모드 복귀

**레이아웃**
- 슬라이드 영역 + 우측 버튼 패널 유지 (슬라이드 모드와 동일)
- dim 오버레이 제거 — 슬라이드를 밝게 표시
- 레이저 버튼 → 활성 색상 (Electric Violet `#7270ff`)

**레이저 동작**
- 슬라이드 영역 터치 위치를 레이저 좌표로 전송
- 1.2초 후 자동 페이드아웃 (PRD §11.3)
- 손가락 이동 시 실시간 좌표 업데이트

---

### 3.4 판서 모드

**진입**: 슬라이드 모드에서 판서 버튼 탭
**종료**: 슬라이드 영역 **더블탭** → 슬라이드 모드 복귀

**레이아웃**
```
┌────────────────────────────────────┐
│                                    │
│         슬라이드 풀페이지           │
│         (우측 버튼 패널 숨김)       │
│                          [🗑 지우기]│ ← 우측 하단 플로팅
└────────────────────────────────────┘
```

- 우측 버튼 패널 **숨김** → 슬라이드가 전체 화면 차지
- 판서 모드 진입 시 "판서 모드 · 더블탭으로 종료" 토스트 짧게 표시

**판서 인터랙션**
- **한 손가락 드래그**: 선 그리기 (Canvas 판서)
  - 정규화 좌표(0.0~1.0)로 변환 후 `draw_event` emit (PRD §11.2)
  - 펜 색상: 코랄 레드 `#FF6B6B` (단일 고정, MVP)
  - 펜 두께: 4px (PRD §15 기본값)
- **두 손가락 핀치**: 줌인 / 줌아웃 (최소 1x / 최대 3x)
- **두 손가락 드래그**: 줌인 상태에서 화면 패닝

**판서 모드 전용 플로팅 버튼**
- **지우기 버튼**: 우측 하단, 원형 플로팅
  - 탭 시 현재 페이지 판서 전체 삭제 (`draw_clear` emit)
  - 확인 없이 즉시 실행

```
슬라이드 모드 (디폴트)
    ├─ 스크립트 버튼 탭 ──→ 스크립트 모드

    └─ 판서 버튼 탭 ──────→ 판서 모드
            └─ 더블탭 ──────────────→ 슬라이드 모드
```

## 4-1. Radial Menu (롱프레스 빠른 메뉴)

슬라이드 영역을 **롱프레스**하면 누른 지점을 기준으로 **독립적인 원형 버튼 3개**가 위쪽 방향으로 펼쳐진다. 손가락을 떼지 않고 원하는 버튼으로 **드래그**해 선택하고, **손가락을 떼면** 해당 메뉴가 실행된다. (판서 모드에서는 그리기와 충돌하므로 비활성)

**트리거 / 동작**

- 트리거: 슬라이드 영역 롱프레스(약 450ms)
- 형태: 독립 원형 버튼 3개가 롱프레스 지점 기준 **위쪽으로 펼쳐짐**
- 버튼 크기: **70px × 70px 원형** (64px의 1.1배)
- 버튼 간격: 엄지로 누르기 편하도록 충분히 떨어뜨림
- 선택: 손가락을 떼지 않고 드래그 → 가리키는 버튼 하이라이트
- 실행: 손가락을 떼면 선택된 버튼 실행 (어느 버튼에도 닿지 않으면 취소)

**버튼 3개 배치 (롱프레스 지점 기준)**

| 위치 | 메뉴 | 동작 |
|------|------|------|
| 상단 (12시) | 슬라이드 | 슬라이드 모드로 복귀 |
| 우상단 (2시) | Q&A | Q&A 화면으로 전환 |
| 좌상단 (10시) | 발표 끝 | "발표를 종료할까요?" 컨펌 후 세션 종료(`presentation_end`) |

**스타일**

- 기본 버튼: Dark Surface `#1c1a2c` 배경, 텍스트 Paper `#ffffff`, 14px, weight 500
- 호버/선택 중: Electric Violet `#7270ff`
- "발표 끝" 텍스트: 코랄 레드 `#FF6B6B` (위험 액션 강조)
- 등장 애니메이션: scale + opacity, 150ms

**발표 끝내기 컨펌 다이얼로그**

- 문구: "발표를 종료할까요?"
- 확인(종료) 버튼: 코랄 레드 `#FF6B6B` 배경
- 취소 버튼: Dark Surface `#1c1a2c` 배경, Silver `#9ca3af` 텍스트

## 4-2. Q&A 화면

Radial Menu에서 Q&A를 선택하면 진입하는 화면. 청중이 제출한 질문을 보고, 특정 질문을 선택해 전체 화면에 강조(팝업)한다.

**화면 구성 (가로 모드 기준)**

```
┌──────────────────────────────────────────┐
│ ← 돌아가기        Q&A              3개      │ ← 상단 바
├──────────────────────────────────────────┤
│ ┌──────────────────────────────────────┐ │
│ │ 질문 카드 (오래된 순, 새 질문은 하단) │ │
│ └──────────────────────────────────────┘ │
│ ┌──────────────────────────────────────┐ │
│ │ ...                                  │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

- **상단 바**: 좌측 `← 돌아가기`(슬라이드 모드 복귀), 중앙 `Q&A` 타이틀, 우측 총 질문 수
- **본문**: 질문 카드 리스트, **오래된 순** 정렬, 새 질문은 **하단에 추가**

**질문 카드**

- 카드 형태: Dark Surface `#1c1a2c` 배경, Mist `#e5e7eb` 1px 보더, border-radius 8px, padding 16px
- 질문 텍스트: Paper `#ffffff`, 16px, weight 400
- 비활성 상태: 기본 카드 스타일

**활성 상태 (탭 시)**

- 카드 배경 전체를 Electric Violet `#7270ff` 으로 변경
- 질문 텍스트: Paper `#ffffff`
- 보더 제거
- 우측 상단 확성기 아이콘(📢): Paper `#ffffff`
- 다시 탭 → 비활성화, 확성기 아이콘 사라짐, 팝업 해제(`qa_highlight isVisible:false`)
- **한 번에 하나의 질문만 활성화 가능**

**이벤트**

- 질문 수신: `question_added` (Server → Presenter)
- 강조 토글: `qa_highlight { questionId, isVisible }` (Presenter → Server → Display/Audience)

**후속 (미구현)**

- AI 중복 질문 병합 및 인기순 정렬

## 7. 비기능 요건

- 리모컨 화면 진입 시 **Screen Wake Lock** 자동 요청 (PRD §12.3)
- 화면 방향 전환 시 Canvas 판서 상태 유지
- 줌 상태에서 슬라이드 넘기기 **비활성** (줌 해제 후 넘기기)
- 판서 모드에서 슬라이드 넘기기 **비활성** (더블탭으로 모드 종료 후 넘기기)
